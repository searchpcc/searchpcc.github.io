MacOS ClashX Pro 解决配置文件覆盖的方法

背景 由于自己在某种特殊场景下需要屏蔽某类带有某个关键字的域名。所以涉及到自己定义 Clash 的规则，而直接在订阅的机场的 YAML 文件内进行修改的话，每次更新配置文件之后原有的修改就会被覆盖。所以就涉及到自己编写属于自己的配置文件，实现自定义规则。并且需要保证机场的节点最新信息能够自动更新。 基础配置 Clash 的配置文件主要由两大部分组成：代理（Proxy）和规则（Rules）。 配置 Proxy 1、配置 proxy-providers（代理集） path：是通过自定义配置拉取的机场的配置文件存放的路径 url：填写机场的订阅地址 health-check：是健康状态选择 interval：自动更新时间，单位为秒 filter：按照原机场对于节点描述进行过滤。如果原来叫 Hong Kong 则不能填写 香港 ，而要填写 Hong Kong proxy-providers: AgentNEO-HongKong: type: http path: ./ProxySet/AgentNEO/HongKong.yaml url: "机场订阅地址" interval: 1800 filter: '香港' health-check: enable: true url: http://www.gstatic.com/generate_204 interval: 300 AgentNEO-Singapore: type: http path: ./ProxySet/AgentNEO/Singapore.yaml url: "机场订阅地址" interval: 1800 filter: '新加坡' health-check: enable: true url: http://www.gstatic.com/generate_204 interval: 300 2、配置 proxy-groups（策略组） type：select 是手动选择，url-test 是自动选择，自动选择是根据状态检测时长的长短来优先选择 name：是定义可选择模式 use：是设定使用的 proxy-providers 的选项 - name: 手动选择 type: select url: http://www....
